## 一、项目概述

### 1.1 核心定位

| 维度 | 设计 |
|------|------|
| **使用场景** | 个人决策前的快速检索与推演参考 |
| **输入方式** | 自然语言描述当前处境（支持语音输入） |
| **输出方式** | 匹配案例 + 风险警示 + 行动检查清单 |
| **数据主权** | 完全本地存储，无需联网（可选增强检索时联网） |
| **扩展性** | 支持个人持续录入新案例，形成私有知识库 |

### 1.2 核心价值主张

> **"让过去的智慧照亮未来的决策"**

系统通过结构化存储个人历史决策案例，结合AI增强的检索与推演能力，帮助用户：
- 避免重复犯错
- 发现隐藏的模式
- 跨领域迁移经验
- 校准认知偏见

## 二、系统架构

### 2.1 整体架构图

```
┌─────────────────────────────────────────┐
│           交互层（TUI/Web）              │
│      输入：自然语言 → 输出：结构化报告      │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│           核心引擎层（Python）            │
│  ┌─────────┐ ┌─────────┐ ┌───────────┐ │
│  │ 语义理解 │→│ 混合检索 │→│ 因果推演   │ │
│  │ (轻量)  │ │ (向量+图)│ │ (规则+LLM) │ │
│  └─────────┘ └─────────┘ └───────────┘ │
└─────────────────────────────────────────┘
                    │
┌─────────────────────────────────────────┐
│           数据层（全本地）               │
│  SQLite(结构化) + FAISS(向量) + JSON(原文) │
└─────────────────────────────────────────┘
```

### 2.2 技术栈选型

| 层级 | 组件 | 选型 | 理由 |
|------|------|------|------|
| **运行环境** | Python版本 | 3.10+ | 稳定，生态成熟 |
| | 包管理 | `uv` 或 `pdm` | 比pip快10倍，锁定依赖 |
| **数据存储** | 结构化数据 | SQLite | 零配置，单文件，支持JSON字段 |
| | 向量索引 | `faiss-cpu` | Meta出品，无需GPU，百万级毫秒检索 |
| | 文档存储 | 纯JSONL文件 | 案例原文直接存，版本控制友好 |
| | 图关系 | `networkx` | 案例关联图谱，轻量无依赖 |
| **语义理解** | 文本嵌入 | `sentence-transformers` + `paraphrase-multilingual-MiniLM-L12-v2` | 多语言支持（中英），118MB，本地运行 |
| | 意图分类 | 规则+轻量分类器 | 无需大模型，识别查询领域 |
| | 实体提取 | `spaCy` (zh_core_web_sm/en_core_web_sm) | 本地NER，识别人物/公司/时间 |
| **检索引擎** | 混合检索 | 向量相似度 + 约束条件过滤 + 图谱关联 | 三层召回，精准匹配 |
| | 重排序 | 轻量交叉编码器或规则评分 | 提升Top-K准确率 |
| **推演生成** | 因果推理 | 预定义规则模板 + 图谱路径搜索 | 解释性强，无需训练 |
| | 文本生成 | 本地小模型(`phi-3`/`qwen2-1.5b`)或云端API可选 | 生成检查清单解释 |
| **交互界面** | 主界面 | `textual` (TUI) | 终端界面，键盘操作，极客友好 |
| | 备选 | `gradio` | 如需浏览器界面，5分钟搭建 |
| | 语音输入 | `whisper.cpp` (本地) 或系统自带 | 快速录入想法 |
| **辅助工具** | 数据采集 | `crawl4ai` + `playwright` | 自动化抓取公开案例 |
| | 数据标注 | 自制TUI表单 | 结构化录入新案例 |
| | 同步备份 | `rclone` 或 `git` | 私有云备份知识库 |

## 四、核心功能模块

### 4.1 混合检索策略

```
用户输入："我现在是一个A轮公司CTO，技术债很重，想重构但业务压力大"

          ↓
    ┌─────────────┐
    │  意图解析    │ → 识别领域：tech + business 混合
    │  (spaCy)    │ → 提取实体：A轮、CTO、技术债、重构
    └─────────────┘
          ↓
    ┌─────────────┐
    │  多层召回    │
    └─────────────┘
         │
    ┌────┴────┬────────┬────────┐
    ↓         ↓        ↓        ↓
 向量检索   约束过滤   图谱关联   时序匹配
(语义相似) (A轮+CTO) (技术债→重构) (近期案例优先)
    │         │        │        │
    └────┬────┴────────┴────────┘
         ↓
    ┌─────────────┐
    │  融合排序    │ → 综合评分，去重
    │  (规则权重)  │
    └─────────────┘
         ↓
    Top 5 案例 + 因果链可视化
```

### 4.2 因果链自动挖掘

三层挖掘架构：

```
┌─────────────────────────────────────────────────────────┐
│  Layer 3: 高阶因果推理（图神经网络）                        │
│  ─────────────────────────────────                      │
│  输入: 初步因果图 + 案例库                                │
│  输出: 补全的因果边 + 因果强度 + 潜在因果路径               │
│  技术: GNN链接预测 + 反事实模拟                            │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│  Layer 2: 结构化因果抽取（规则+统计）                       │
│  ─────────────────────────────────                      │
│  输入: 分句后的案例描述                                   │
│  输出: (因→果) 关系对 + 关系类型 + 置信度                  │
│  技术: 依存句法分析 + 因果模板匹配 + PMI统计               │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│  Layer 1: 因果感知分句与实体识别                            │
│  ─────────────────────────────────                      │
│  输入: 原始案例文本                                       │
│  输出: 候选因果句 + 标注的因果实体                          │
│  技术: spaCy NER + 因果触发词识别                          │
└─────────────────────────────────────────────────────────┘
```

### 4.3 跨领域相似度计算

四层相似度模型：

```
┌─────────────────────────────────────────────────────────┐
│  Level 4: 元结构相似（最高层）                             │
│  ───────────────────────────                            │
│  抽象为: 系统动力学模型 / 博弈论模型 / 复杂网络特征          │
│  匹配: 反馈回路结构 / 均衡点性质 / 相变临界点               │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│  Level 3: 因果拓扑相似                                    │
│  ─────────────────                                      │
│  抽象为: 因果图结构（忽略实体具体内容）                     │
│  匹配: 因果链长度 / 汇聚/发散节点 / 反馈环存在性            │
└─────────────────────────────────────────────────────────┘
                           ↑
┌─────────────────────────────────────────────────────────┐
│  Level 2: 约束条件相似                                    │
│  ─────────────────                                      │
│  抽象为: 决策情境的特征向量                                │
│  匹配: 时间压力 / 资源限制 / 信息完整度 / 利益相关方数量      │

                           ↑

│  Level 1: 实体语义相似（基础层）                            │
```

### 4.4 AI"红队"对话机制

**核心设计：AI作为专业反对者**

分层攻击流程：

```
Layer 1: 事实基础攻击
"你的理由A基于'市场正在增长'，数据来源是？
 我找到3个相反信号：[列举]
 如果市场实际萎缩，你的推理还成立吗？"

Layer 2: 逻辑结构攻击  
"你的推理：A→B→X
 但历史上，A→B的转化率只有30%（基于你的案例库）
 且B→X需要假设C成立，而C在你的情境中不成立，因为..."

Layer 3: 隐含假设攻击
"你假设'团队能执行'，但回顾你的案例#12、#23，
 类似情境下团队执行失败率60%。
 这次有什么不同？具体说3点。"

Layer 4: 极端情境攻击
"假设最坏的10%情况发生（基于历史分布），
 你的预案是什么？
 [生成具体极端情境]
 你现在还愿意继续吗？"

Layer 5: 机会成本攻击
"如果你不做X，资源可投入Y或Z。
 基于你的历史，Y的成功率是X的1.5倍。
 你确定X是最佳选择，而非只是最熟悉的选择？"
```

### 4.5 "未来你"对话机制

**核心设计：时间透视的折叠**

```
成功未来你（1年后，决策X结果良好）：
• 状态：自信、感恩、有掌控感
• 关键行动：做对了什么？
• 转折点：哪个时刻最关键？
• 给现在的建议：坚持什么？

失败未来你（1年后，决策X结果糟糕）：
• 状态：后悔、疲惫、资源耗尽
• 关键失误：忽略了什么信号？
• 转折点：什么时候应该停止？
• 给现在的建议：改变什么？

中立未来你（5年后，已淡忘细节）：
• 状态：平静，将其视为人生一课
• 长期影响：这个决策如何改变了你？
• 本质洞察：当时真正重要的是什么？
```

## 五、关键设计原则

### 5.1 核心取舍

| 追求 | 放弃 | 理由 |
|------|------|------|
| 深度而非广度 | 全领域覆盖 | 个人使用，垂直穿透更有价值 |
| 可解释性 | 预测精度 | 你要的是理解，不是黑箱答案 |
| 结构化输入 | 自然语言自由 | 强制你清晰思考，而非模糊感觉 |
| 本地离线 | 实时联网增强 | 数据主权，且防止依赖外部 |
| 长期积累 | 即时满足 | 价值在3-5年后才充分显现 |

### 5.2 关键设计原则总结

| 原则 | 实现 |
|------|------|
| **你的数据，你的控制** | 本地优先，加密存储，可选联邦 |
| **AI增强，而非替代** | 你始终拥有最终决定权和责任 |
| **持续进化，非一次性** | 每次对话都是系统学习机会 |
| **效率与深度平衡** | 自适应压缩，按需加载 |
| **透明可信** | AI展示推理过程，你验证和修正 |
| **长期主义** | 3年积累，10年受益，终身成长 |

### 5.3 人机协作边界

```
初期（案例<50）：
• AI主导：提供框架，你填充内容
• 你学习：理解结构化思考方式

中期（案例50-200）：
• 协作：AI提供选项，你选择并修正
• 共同发现：AI识别模式，你验证意义

后期（案例>200）：
• 你主导：AI成为你的"外部记忆"
• AI适应：学习你的独特风格，个性化增强
• 双向进化：你优化AI，AI优化你

关键原则：
AI的"主见"随你的案例积累而减少
初期AI多引导，后期AI多支持
最终目标：AI隐形，但你决策质量显著提升
```

## 六、演进路线

### 阶段一：习惯养成（0-3月）

**核心目标**：建立决策记录习惯，不追求分析质量

- 每周至少1次重大决策使用系统
- 每次记录不超过10分钟
- 强制字段：选择了什么，为什么，预测结果

### 阶段二：模式积累（3-12月）

**核心目标**：发现个人决策模式，建立基线

- 开始输出简单统计（各领域胜率）
- 提示重复出现的决策情境
- 推荐历史相似案例

### 阶段三：深度校准（1-2年）

**核心目标**：系统成为你的"外部自我"，能预警你的盲区

- 偏见实时检测与干预
- 个人决策理论生成与验证
- 复杂情境的因果推演

### 阶段四：共生进化（2年+）

**核心目标**：系统持续进化，你持续超越过去的自己

- 系统开始提出你从未想过的类比
- 你发现系统模式的局限，反哺优化
- 形成独特的个人决策方法论
